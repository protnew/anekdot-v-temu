## Сценарий 1. Веб-интерфейс (SPA)

**Как запустить:** `start.bat` → [1] или `python launcher.py server`

**Что происходит:**
1. Пользователь открывает `http://localhost:8000` в браузере
2. Видит тёмный интерфейс с полем ввода и кнопками
3. Вводит тему (например, «программист» или «муж и жена»)
4. Сервер через TF-IDF (5000 признаков) ищет топ-3 похожие шутки из 81K базы
5. Показывает шутку + категорию + рейтинг + процент совпадения

**Доступные действия в вебе:**
- 🔍 Поиск по теме (контекстный семантический поиск)
- 🎲 Случайная шутка (из всей базы)
- 📂 Фильтр по категории (21 категория)
- ⭐ Рейтинг (1–5 звёзд)
- ❤️ Избранное (localStorage + API)
- 🔊 Озвучить шутку (gTTS → MP3)
- 📋 Скопировать в буфер
- 🤖 AI-генерация (если есть OpenAI ключ)
- 📊 Аналитика популярных запросов
- 📱 PWA (установка на телефон как приложение)

**API endpoints:**
```
GET  /                          — SPA интерфейс
GET  /api/joke/random           — случайная шутка
GET  /api/jokes?category=айти   — шутки по категории
GET  /api/jokes/search?q=тема   — текстовый поиск
POST /api/jokes/context          — семантический поиск
POST /api/jokes/generate         — AI-генерация (GPT-4o-mini)
POST /api/rate                   — оценить шутку
POST /api/favorites              — добавить в избранное
GET  /api/favorites              — мои избранные
GET  /api/categories             — список категорий с кол-вом
GET  /api/jokes/social/top       — топ шуток дня
POST /api/jokes/{id}/like        — лайкнуть шутку
POST /api/user-jokes             — добавить свою шутку
GET  /api/user-jokes             — мои шутки
GET  /api/analytics/popular      — популярные запросы
GET  /api/analytics/stats        — статистика поиска
```

## Сценарий 2. Голосовой ассистент (микрофон → шутка)

**Как запустить:** `start.bat` → [2] (base) или [3] (small) или `python launcher.py voice`

**Что происходит:**
1. Voice Monitor (`voice_monitor.py`) захватывает микрофон
2. Записывает аудио порциями по 10 секунд (VAD — Voice Activity Detection)
3. Whisper (base 138MB или small 461MB) распознает русскую речь
4. Fallback на Google Speech Recognition если Whisper не установлен
5. Распознанный текст отправляется на `POST /api/jokes/context`
6. Лучший результат показывается:
   - В консоли (голосовой режим)
   - В overlay на экране (полный режим)

**Параметры:**
- `WHISPER_MODEL` env var: `base` (быстрый, 138MB) или `small` (качественный, 461MB)
- `BASE_URL` env var: адрес API сервера (по умолчанию `http://localhost:8000`)

**API endpoints:**
```
POST /api/voice/stt  — Speech-to-Text (загрузить аудио → получить текст)
```

## Сценарий 3. Overlay (шутки поверх всех окон)

**Как запустить:** `start.bat` → [4] или `python launcher.py full`

**Что происходит:**
1. `overlay.py` создаёт полупрозрачное окно tkinter поверх всех окон
2. Окно появляется в правом нижнем углу экрана
3. Показывает шутку + категорию + рейтинг
4. Автоматически исчезает через 15 секунд
5. Новая шутка появляется при каждом распознанном фрагменте речи
6. Можно закрыть крестиком или нажать Esc

**Особенности overlay:**
- Всегда поверх других окон (topmost)
- Полупрозрачный фон (можно читать текст под ним)
- Плавное появление/исчезновение (alpha fade)
- Автоподстройка под длину шутки

## Сценарий 4. Telegram бот

**Как запустить:** `TELEGRAM_BOT_TOKEN=... python bot/telegram_bot.py`

**Команды бота:**
| Команда | Описание |
|---|---|
| `/start` | Приветствие + список команд |
| `/random` | Случайный анекдот из 81K |
| `/categories` | Все 21 категория с количеством |
| `/top` | Топ анекдотов дня |
| `/stats` | Статистика базы (81K, 21 кат.) |
| `/cat работа` | Анекдоты из конкретной категории |
| Любой текст | Контекстный подбор шутки |

**Inline mode (в любом чате):**
1. Пишешь `@bot_имя программист`
2. Появляется список из 5 подходящих шуток
3. Выбираешь → шутка отправляется в чат
4. Работает в ЛЮБОМ чате, даже где бот не добавлен

**Особенности:**
- Каждая шутка показывает категорию + рейтинг + процент совпадения
- Если не нашёл — предлагает случайную
- Роутер команд через `set_my_commands` (меню в Telegram)

## Сценарий 5. Docker / VPS деплой

**Как запустить:** `bash deploy.sh` на сервере с Docker

**Что происходит:**
1. Создаёт `.env` файл с ключами (OpenAI, Telegram токен)
2. Билдит Docker образ (`python:3.12-slim` + `ffmpeg` + все зависимости)
3. Запускает два контейнера:
   - **api** — FastAPI сервер на порту 8000, healthcheck каждые 30 сек
   - **bot** — Telegram бот, зависит от api (ждёт healthcheck OK)
4. Автоматический restart при падении (`unless-stopped`)
5. Лимит памяти: API — 2GB, бот — 512MB

**Docker Compose сервисы:**
```yaml
services:
  api:                          # Основной сервер
    ports: 8000
    healthcheck: /api/stats
    memory: 2G
    
  bot:                          # Telegram бот  
    command: python bot/telegram_bot.py
    depends_on: api (healthy)
    memory: 512M
```

**Команды управления:**
```bash
bash deploy.sh                          # Запуск
docker compose logs -f                  # Логи
docker compose restart                  # Рестарт
docker compose down                     # Стоп
```

## Сценарий 6. AI-генерация шуток

**Как запустить:** Автоматически при наличии `OPENAI_API_KEY`

**Что происходит:**
1. Пользователь вводит тему в веб-интерфейсе
2. `POST /api/jokes/generate` отправляет промпт в GPT-4o-mini
3. Промпт: «Придумай один короткий русский анекдот на тему: ...»
4. Ответ парсится и возвращается с пометкой `generated: true`
5. Fallback: если API ключа нет — возвращает шаблонные шутки

**Без OpenAI ключа:**
- Контекстный поиск через TF-IDF работает (81K база)
- Генерация новых шуток недоступна
- Все остальные функции работают полностью

## Сценарий 7. Веб-логи и мониторинг

**Как запустить:** Автоматически вместе с сервером

**Что доступно:**

**Страница логов (`/logs`):**

## Сценарий 8. Персонализация

**API endpoints:**
```
POST /api/personalize/{user_hash}   — Сохранить предпочтения
GET  /api/personalize/{user_hash}   — Получить предпочтения
```

**Как работает:**
1. На основе истории лайков и оценок формируется профиль
2. Любимые категории получают повышенный вес при поиске
3. Непонравившиеся темы — пониженный вес
4. Хранится в SQLite (`data/jokes.db`)

## Сценарий 9. TTS — Озвучка шуток

**API endpoints:**
```
POST /api/voice/tts              — Сгенерировать MP3 из текста
GET  /data/tts/{filename}        — Скачать MP3 файл
```

**Как работает:**
1. В веб-интерфейсе кнопка 🔊 на каждой шутке
2. gTTS (Google Text-to-Speech) генерирует MP3
3. Файл сохраняется в `data/tts/`
4. Воспроизводится через `<audio>` в браузере
5. Работает полностью бесплатно, без API ключей
